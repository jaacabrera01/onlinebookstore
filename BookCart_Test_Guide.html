Think of testing like quality checking at a factory:

Normal Factory Process:
Make Product → Test It → Check If It Works → Ship It
BookCart Testing:
Write Code → Run Tests → Check If It Works → Fix If Needed
Current Problem:

This is like trying to test a store with no products on the shelves!

The Actual Situation:

  • We have a bookstore website (bookcart.azurewebsites.net)
  • The website currently has NO BOOKS in the database
  • Our tests try to add books to cart, but there are no books!
  • So the tests fail automatically

What We Need to Do:

  • Add some sample books to the bookstore
  • Then the tests can run properly
  • Then we'll see which tests actually work

📊 Section 8: What Each Result Means

When Tests Finish, You'll See Results:

✅ PASSED

Meaning: This test worked perfectly!
What it means: The feature is working correctly
Action: No action needed - celebrate!

Example: ✅ test_login_with_valid_credentials PASSED
❌ FAILED

Meaning: This test did not work
What it means: There's a problem with this feature
Action: Make note of the error and report it

Example: ❌ test_add_to_cart FAILED - Error: Element not found
⊘ SKIPPED

Meaning: This test was not run
Why: Usually because test data is missing (expected)
Action: This is NORMAL - no action needed

Example: ⊘ test_featured_books_displayed SKIPPED - Reason: No books available in the database

Final Summary Line:

You'll see something like:

=============== 34 passed, 5 skipped, 18 failed in 479.90s ================

This means:

  • 34 tests worked fine ✅
  • 5 tests were skipped (not run, expected) ⊘
  • 18 tests failed ❌
Good Goal to Aim For:
  • More PASSED than FAILED
  • Some SKIPPED is normal
  • 34+ PASSED is excellent

🔧 Section 9: Troubleshooting

Problem: "Command not found: pytest"

Solution 1: You didn't activate the environment
Type: source .venv/bin/activate
Try pytest command again
Solution 2: You're not in the right folder
Type: pwd (to see current location)
Should show: /Users/jaacabrera/Documents/Python Scripts/Online Bookstore
If not, navigate there first

Problem: Tests are taking very long (more than 20 minutes)

This is normal for full test suite!

  • Expected time: 8-15 minutes
  • To speed up, run just specific tests: pytest tests/test_authentication.py -v

Problem: "Still on login page after login"

This usually means the website credentials don't work. Solution:

  1. Verify you can login manually at bookcart.azurewebsites.net
  2. Check that your credentials are correct
  3. Contact system administrator if login doesn't work

Problem: "Element not found" errors

This means the test code is looking for something on the website that isn't there.


Possible causes:
  • Website design changed
  • Books aren't in the database (back to Phase 2!)
  • Form fields are in different places
Solution:
  • Make sure you completed Phase 2
  • Check that books are visible on the home page
  • Contact your developer team

Problem: "No books available to add to cart"

This is actually EXPECTED and OK!

Meaning: You haven't added test data yet
Solution: Go back to Phase 2 and add books

This is NOT a test failure - it's a SKIPPED test
